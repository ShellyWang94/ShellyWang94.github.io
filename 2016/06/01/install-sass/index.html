    • 一.安装方法

    (一)安装ruby

    直接在官网上下载安装包，点击下一步，下一步，下一步，记得勾选选项的时候选择“add Ruby executables to your PATH”

    (二)安装完成后：

    在程序中搜索ruby，选择start Command prompt with Ruby应用（其实直接在cmd中运行也行）

    (三)安装Sass

    在网上的依靠ruby的安装方法一共有五种，这里先罗列一下：

    1. 通过命令gem install sass来安装

    2. 通过github仓库来安装

    git clone git://github.com/nex3/sass.git
    cd sass
    rake install

    3. 通过本地安装

    这点需要先到https://github.com/sass/sass去下载了安装包，放在本地，之后在命令行中输入 gem install 安装包路径

    4. 淘宝RubyGems镜像安装Sass

    先移除默认https：//rubygems.org源
    添加淘宝源http://rubygems-china.oss-cn-hangzhou.aliyuncs.com

    5. 以及官网上所述的无需ruby的安装方法


    http://sass-lang.com/install
    对啦，官网上的安装软件方法应该是最简单的，然而我并没有去试，因为它们都是基于sass开发的框架，所以安装好它们也相当于安装好sass了

    以上五种是我在安装当中搜索到的方法，官网上对于安装ruby环境然后使用cmd安装有详细的步骤

    官网链接：HTTP://WWW.SASS.HK/INSTALL-SASS.HTML

    二.但是，一切总是要出问题的/(ㄒoㄒ)/

    （一）要谈问题，就从第一个方法的问题谈起吧~

    安装ruby的过程不能再简单，我满心欢喜地以为能在很快的时间内安好sass，然而事实证明是我想多了
    当我输入gem install sass出现的是酱紫的场景

    用我糟糕的口语来翻译一下大意就是网络错误
    excuse me？
    呵呵，原来是国外的源，国内的墙，这堵墙挡了连接获取源文件啊
    但是，我是不会轻易放弃地，于是，我开启latern再次来一次，然并卵，依然是这样的效果，我想哭，但是最简单的安装方式是不行的了

    （二）于是转战本地文件安装


    可能是我输入错误还是怎么的，反正就是不行。在这里在说一个小插曲，我直接给了github上的链接，因为我在sass主页上徘徊了很久点了很久链接依然点不动，最后在文档哪里找到了，总之，这种方法依然错误，依然没有成功

    （三）然而我还不能狗带，带着决心开启了淘宝镜像安装

    当我移除了源地址（gem sources –remove https：rubygems.org）之后，再次设置淘宝源地址——即官网给出来的（gem sources a- https://ruby.taobao.org）时

    我一副大写的生无可恋脸（忘说，http和https我都试过，一样的效果，因为淘宝已经不维护了，换地址了，即开头所写的地址）
    好了，到这里为止我都觉得老天一定是在玩我
    最后经过九牛二虎之力，我终于找到了正确的地址安装上了sass，地址在开头已经附上~
    对了，在检测源地址是否已经改变的时候可以输入gem sources -l命令，如果成功，会显示你设置的新的源地址
    放上我成功的截图

    输入sass -v，没问题哒，正确显示

    血泪提示：

    1. 当remove之后，如果设置不上的话，那么想再试试第一种是一定会出错（似乎是找不到sass的意思)

    2. 如果你的电脑第一种方法安装不上的话，那么源地址也无法设置回去，这时候我采取的方法是将ruby卸载，卸载完成之后把c盘里的源文件删光十分重要（一般是在user>ausus下的，看你的命令在哪里运行），否则重新安装仍然是一样的配置

    3. 教程建议我安装ruby在c盘，我安装在D盘，卸载的时候删光了所有相关文件，得到干净的环境
